# 관리자센터 유저 권한 체계

출처: us-admin 소스코드 (`useSidebar.ts`, `sidebar.ts`)

---

## 역할 (Role)

| Role | 설명 | 메뉴 접근 |
|------|------|----------|
| `ALL` | 슈퍼 관리자 | 모든 메뉴 |
| `PART` | 부분 관리자 | 할당된 권한 섹션만 |
| `NONE` | 권한 없음 | 메뉴 없음 |

## 권한 섹션 (permissionSections)

| 권한 | 메뉴명 | 설명 |
|------|--------|------|
| `DASHBOARD` | 대시보드 | 주요 지표, 마스터, 회원, 매출, 구독, 콘텐츠 관리 |
| `USER` | 회원관리 | 회원 정보 조회, 연동상품 구매자 관리 |
| `SALES` | 매출관리 | |
| `SUBSCRIPTION` | 구독관리 | |
| `PRODUCT` | 상품관리 | 상품 관리, 상품 결제 내역 관리, 연동 상품 관리 |
| `OPERATION` | 서비스 운영 관리 | 배너 관리, 라운지 관리 등 |
| `CONTENT` | 콘텐츠 관리 | 콘텐츠 조회 관리 등 |
| `CS` | CS 관리 | 고객 지원, 환불 관리 등 |
| `SHOP` | Shop 관리 | 상품 관리, 주문 관리 등 |
| `MASTER` | 마스터 관리 | 슈퍼 관리자 전용 |
| `PERMISSION` | 권한 관리 | 슈퍼 관리자 전용 |

## 메뉴 필터링 로직

핵심 파일: `src/shared/model/hooks/useSidebar.ts`

- `ALL`: SIDEBAR_CONFIG 전체 반환
- `PART`: `requiredPermissions`와 유저의 `permissionSections` 교집합이 있는 메뉴만 표시
- `NONE`: 빈 배열

## "마스터 & 권한관리" 섹션 특수 케이스

| 하위 메뉴 | 필요 권한 |
|-----------|----------|
| 운영 권한 | `PERMISSION` |
| 서비스 이용 권한 관리 | `PERMISSION` |
| 오피셜 클럽 관리 | `PERMISSION` |
| 마스터 관리 | `MASTER` |

## 데이터 흐름

```
로그인 → AuthService가 role + permissionSections 저장
  → SideBarProvider가 읽어서 useSidebar()로 필터링
  → AdminAppSidebar가 필터된 메뉴 렌더링
```

저장 위치:
- `role`: localStorage (userInfo)
- `permissionSections`: sessionStorage

## JWT 토큰 구조

```json
{
  "role": "SUPER",           // 또는 "ALL" / "PART" / "NONE"
  "roleType": "ALL",
  "permissionSections": [],  // PART일 때 ["PRODUCT", "MASTER"] 등
  "email": "...",
  "name": "...",
  "department": "..."
}
```
