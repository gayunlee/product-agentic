# AI 어시스턴트가 알아야 할 도메인 지식

## 0. 핵심 데이터 구조 — 마스터 그룹 / 오피셜클럽 / 상품

```
마스터 그룹 (/v1/master-groups)          오피셜클럽 (/v1/masters)
┌───────────────────┐             1:N   ┌──────────────────────────┐
│ id: "69b95eff"    │◄──────────────────│ masterGroupId: "69b95eff"│
│ name: "조조형우"  │                   │ id (MongoDB _id)         │
│ masterCount: 1    │                   │ cmsId: "35" (= masterId) │
└───────────────────┘                   │ name: "조조형우"         │
                                        │ publicType: PUBLIC       │
                                        └──────────┬───────────────┘
                                                   │ 1:N
                                        ┌──────────┴───────────────┐
                                        │ 시리즈 (파트너센터)       │
                                        │ 상품 페이지              │
                                        │   └─ 상품 옵션           │
                                        │ 게시판                   │
                                        │ 응원하기                 │
                                        │ 편지글                   │
                                        └──────────────────────────┘
```

### 관계 정리

| 개념 | API | ID | 역할 |
|------|-----|-----|------|
| 마스터 그룹 | `GET /v1/master-groups` | `group.id` | 상위 그룹 (이름만). 마스터 관리(/master)에서 생성 |
| 오피셜클럽 | `GET /v1/masters` | `cmsId` (= masterId) | 실제 운영 단위. 마스터 그룹 하위로 생성. masterGroupId로 연결 |
| 상품 페이지 | `POST /v1/product-group` | `productGroupId` | 오피셜클럽의 결제 접점. masterId = cmsId |

### 오피셜클럽 생성 시
- `GET /v1/master-groups` → 드롭다운에 마스터 그룹 목록 표시
- `masterGroupId = group.id`를 선택 → 해당 마스터 그룹 하위에 오피셜클럽 생성
- 오피셜클럽 생성: `/official-club/create`

### 상품 페이지 생성 시
- `GET /v1/masters` → "마스터 선택" 드롭다운 (실제로는 오피셜클럽 선택!)
- `master.name` 표시, `master.cmsId`를 `masterId`로 사용
- **주의**: UI에 "마스터 선택"이라고 표시되지만, 실제로는 오피셜클럽을 고르는 것

### 용어 주의: "마스터" vs "오피셜클럽"

**역사적 배경**: 원래 마스터가 곧 운영 단위였다가, 1:N 구조로 확장되면서 마스터가 상위 그룹으로 올라가고 오피셜클럽이 생김. 그래서 API/코드에 `master`가 남아있음.

**관리자센터(us-admin)에서는 구분됨:**

| 페이지 | 다루는 것 | API |
|--------|----------|-----|
| 마스터 관리 (/master) | 마스터 그룹 (상위) | `/v1/master-groups` |
| 오피셜클럽 관리 (/official-club) | 오피셜클럽 (하위, 실제 운영 단위) | `/v1/masters` |
| 상품 페이지 생성 (/product/create) | "마스터 선택" = 오피셜클럽 선택 | `/v1/masters` |

**어스플러스(us-plus)에서는 구분 없음 — 전부 오피셜클럽 단위:**
- 멤버십 구독 탭: `GET /v1/masters/product-group` → 오피셜클럽별 상품페이지 목록
- 마스터 추천 탭: `GET /v1/masters?status=ACTIVE` → ACTIVE 오피셜클럽 목록, 클릭 시 `/club/{master.id}` = 오피셜클럽 페이지
- 고객 화면에 "마스터 그룹"은 노출되지 않음. 고객이 보는 건 전부 오피셜클럽.

**에이전트 대화 시:**
- 운영매니저가 "마스터"라고 하면 → 문맥에 따라 마스터 그룹 또는 오피셜클럽 (관리자센터 기준)
- "마스터가 안 보여"라고 하면 → 구독 탭에서 해당 오피셜클럽이 안 보인다는 의미 (어스플러스 기준)
- **마스터 그룹(`master-groups`)만 상위 개념이고, API의 `master`는 전부 오피셜클럽을 가리킴**

### 전체 사전조건 체인
```
마스터 그룹 → 오피셜클럽(cmsId) → 시리즈 → 상품 페이지 → 상품 옵션
```
- 마스터 그룹 없음 → 마스터 관리(/master)에서 생성
- 오피셜클럽 없음 → /official-club/create에서 생성
- 시리즈 없음 → 파트너센터(master.us-insight.com)에서 생성
- 상품 페이지/옵션 → 관리자센터에서 생성

## 0-1. 오피셜클럽 필드 → 어스플러스 노출 매핑

### 헷갈리는 3개: name vs displayName vs clubName

| 필드 | 용도 | 어디에 보임 |
|------|------|------------|
| name | 고객 화면 표시명 (최대 17자) | 어스플러스 — 고객이 보는 모든 곳 |
| displayName | 관리자/파트너센터 표시명 | 관리자센터, 파트너센터 내부만 |
| clubName | 오피셜클럽 타이틀 (최대 100자) | 어스플러스 — 클럽 페이지 타이틀 |

name은 짧은 이름(카드, 목록 등), clubName은 긴 타이틀(클럽 페이지 상단) 용도.

### 전체 필드 매핑 (GET /v1/masters/{cmsId})

| 관리자센터 필드명 | DB 키 | us-plus 노출 위치 |
|---|---|---|
| 오피셜클럽명 (고객 표시명) | name | 고객이 보는 이름 전체 |
| 오피셜클럽명 (관리자 표시명) | displayName | us-plus에 안 나옴. 내부용 |
| 오피셜클럽 타이틀 | clubName | 클럽 페이지 상단 타이틀 |
| 오피셜클럽 소개 | clubIntroduction | 클럽 페이지 소개 텍스트 |
| 프로필 이미지 | profileImage | 프로필 아이콘 |
| 클럽 이미지 | clubBackGroundImage | /subscribe 카드 배경 |
| 클럽 텍스트 이미지 | clubTextImage | /subscribe 카드 하단 텍스트 이미지 (마스터 이름 로고) |
| 간단소개 이미지 | simpleIntroductionImage | /subscribe 카드 상단 간단소개 이미지 |
| 오피셜클럽명 이미지 | masterCardTextImage | 코드 참조는 있으나 현재 clubTextImage 사용 중 |
| 간단 소개 | simpleIntroduction | 텍스트 기반 간단소개 (이미지 대체용) |
| 추천 인사말 | recommendIntroduction | 추천 관련 화면 |
| 상세 소개 | detailIntroduction | 클럽 상세 페이지 |
| 대표 이력 | detailCareers | 클럽 상세 페이지 이력 섹션 |
| 편지글 | letter | 클럽 편지 섹션 |
| 편지 알림 메시지 | letterPushMessage | 푸시 알림 (앱) |
| 응원 알림 메시지 | donationPushMessage | 푸시 알림 (앱) |
| 응원하기 타이틀 | donationIntroductionMessage | 응원하기 페이지 소개 |
| 마스터 그룹명 | masterGroupName | us-plus에 안 나옴. 내부용 |
| 커뮤니티 권한 | communityPermissionType | ALL=전체 작성, MEMBERSHIP=멤버십만 |

## 1. 게시판 (Board)

### 관리 (us-admin)

| 경로 | 용도 |
|------|------|
| /board/setting | 게시판 활성/비활성, 이름 변경, 말머리 관리 |
| /board | 게시글 관리 (차단, 고정, 검색) |
| /post/:id | 게시글 상세 (신고 내역) |

### API

| Method | Endpoint | 용도 |
|--------|----------|------|
| GET | /v1/post-boards | 게시판 목록 (마스터별 활성상태) |
| PUT | /v1/post-boards/bulk | 게시판 일괄 수정 {postBoards: [{id, name, activeStatus}]} |
| GET | /v1/posts | 게시글 조회 (masterId, isBlock, nickName, isPinned 필터) |
| PATCH | /v1/posts/:id/block | 게시글 차단/해제 {isBlock} |
| PATCH | /v1/posts/:id/pinned | 게시글 고정 {isPinned} 최대 5개 |
| POST | /v1/post-labels | 말머리 생성 (게시판당 최대 10개, 이름 최대 10자) |

### 모바일웹 표시 조건 (us-plus)

핵심 API: `GET /v1/master/{masterId}/community/category`
- 응답: `{postBoardActiveStatus, postBoardName, donationActiveStatus}`

| 조건 | 화면 |
|------|------|
| postBoardActiveStatus=ACTIVE + 게시글 있음 | 전체 게시판 + 필터 + 목록 + 글쓰기 FAB |
| postBoardActiveStatus=ACTIVE + 게시글 없음 | "아직 작성된 게시글이 없어요" |
| postBoardActiveStatus=INACTIVE | "게시판을 준비하고 있어요" |

탭 표시 규칙:
- 커뮤니티 진입 시: 게시판 활성 → /community/board, 비활성 → /community/letter
- PC: 게시판(isActiveBoard), 응원하기(isActiveDonation), 편지쓰기(항상)
- 모바일: 게시판 탭만 (isActiveBoard일 때)

## 3. 편지글 (Letter)

### 관리 (us-admin)

| 경로 | 용도 |
|------|------|
| /cs/letter | 편지글 관리 (라운지 노출, 차단, 읽음, 순서) |

### API

| Method | Endpoint | 용도 |
|--------|----------|------|
| GET | /v2/letter | 편지 목록 (viewType, blockStatus, readStatus 필터) |
| GET | /v1/letter/:id | 편지 상세 |
| PATCH | /v1/letter/:id/view-type | 뷰타입 변경 {viewType: 'LOUNGE'/'COMMON'/'PRODUCT_GROUP'} |
| PATCH | /v1/letter/view-type/bulk | 일괄 뷰타입 변경 {ids, viewType} |
| PATCH | /v1/letter/sequence/bulk | 라운지 순서 변경 {ids: [순서대로]} |
| PATCH | /v1/letter/block | 차단/해제 {id, isBlock} |
| PATCH | /v1/letter/read/bulk | 읽음 처리 {ids} |

### 편지글 슬라이더 (라운지) 규칙
- viewType === 'LOUNGE' 인 편지만 라운지 슬라이더에 표시
- 최대 8개
- 차단된 편지는 라운지 노출 불가 → "차단된 편지는 라운지에 노출할 수 없습니다."
- 차단 시 라운지에서 자동 제거 (확인 다이얼로그: "차단 시 라운지 노출이 해제됩니다")
- sequence 필드로 표시 순서 관리

### 모바일웹 표시 조건 (us-plus)
- 상품 페이지의 편지글 섹션: `displayLetterStatus === 'ACTIVE'` + 편지 3개 이상 → 탭 표시
- 상품 페이지 고정 편지 슬라이더: viewType === 'PRODUCT_GROUP'으로 조회, 데이터 없으면 자동 숨김
- 편지 API: `GET /v3/letters/by-master/{masterId}` (offset, limit, viewType 파라미터)

## 4. 상품 삭제/노출/히든 규칙

### 삭제 규칙
- 백엔드가 `isDeletable: boolean`을 결정
- `isDeletable === true`일 때만 삭제 버튼 표시
- 활성 구독, 거래 내역 등에 따라 결정

### API

| Method | Endpoint | 용도 |
|--------|----------|------|
| DELETE | /v1/product-group/:id | 상품 페이지 삭제 |
| DELETE | /v1/product/:id | 상품 삭제 |
| PATCH | /v1/product-group/status | 상품 페이지 공개상태 변경 |
| PATCH | /v1/product/display | 상품 표시상태 변경 |

### 노출 우선순위 — "상시" vs "기간 설정"
- **상시** (isAlwaysPublic: true): 마스터당 1개만 가능. 날짜 무관 항상 노출
- **기간 설정** (isAlwaysPublic: false): startAt~endAt 내에서만 노출
- 이미 상시 페이지가 있으면 새 상시 페이지 생성 시 → "상시가 선택된 상품페이지가 이미 존재합니다"

### 히든 vs 비공개 — 다른 개념! ⚠️

| | 히든 (isHidden: true) | 비공개 (status: INACTIVE) |
|--|--|--|
| URL 직접 접근 | **가능** | **불가** ("공개되지 않은 상품 페이지입니다") |
| 구독 탭/오피셜클럽 홈 | 안 보임 | 안 보임 |
| status 값 | ACTIVE 유지 | INACTIVE |
| 기간 충돌 검사 | 제외 | 해당 |
| 용도 | 특정 프로모션/링크로만 접근 | 완전 비공개 (검수 전 등) |

**히든 처리:**
- 생성/수정 시 "히든 처리 여부" 체크박스 → isHidden: true
- 목록에서 숨겨지고 직접 URL로만 접근: `/products/group/{code}`
- status는 ACTIVE 유지 → 백엔드가 URL 접근 허용

**비공개 상태:**
- status: INACTIVE → 백엔드가 URL 직접 접근도 차단
- "공개되지 않은 상품 페이지입니다" 메시지와 함께 fail 반환
- 오피셜클럽이 PUBLIC이든 아니든 상관없이, INACTIVE면 접근 불가

| 오피셜클럽 | 상품페이지 status | URL 접근 결과 |
|--|--|--|
| PUBLIC | ACTIVE | 정상 조회 |
| PUBLIC | INACTIVE | "공개되지 않은 상품 페이지입니다" (차단) |
| PRIVATE | ACTIVE | 마스터 비공개면 목록에 안 뜸 (URL 직접 접근은 별도 확인 필요) |

**상품페이지만 비공개 + 메인/마스터 공개: 오류는 안 남.** 페이지 자체가 INACTIVE이므로 접근 자체 불가.

### 비공개 시 주의사항 (계단식 영향) ⚠️

| 비공개 대상 | 영향 | 주의사항 |
|------------|------|---------|
| 메인 상품페이지 EXCLUDED | 구독 페이지에서 버튼 안 보임 → 상품페이지 접근 불가 | 정상 동작 |
| 상품페이지 INACTIVE | 상품상세 접근 시 "공개되지 않은 상품 페이지" | **모든 페이지가 비공개면 메인도 EXCLUDED 필요** (안 하면 버튼 클릭 시 404) |
| 상품 비공개 (isDisplay=false) | 상품페이지는 뜨지만 상품이 안 보임 | **모든 상품이 비공개면 상품페이지도 비공개 권장** (안 하면 빈 페이지) |

핵심: 하위 레벨이 전부 비공개되면 상위 레벨도 비공개/제외 처리해야 유저에게 404나 빈 페이지가 안 보임.

### "안 보이게 해줘" 의도 구분 ⚠️

"상품페이지를 안 보이게 해줘"는 2가지 의미:
1. **구독 페이지에서 아예 안 보이게** → 메인 상품페이지를 EXCLUDED 처리. 구독 탭에서 버튼 자체가 사라짐.
2. **특정 상품페이지만 비공개** → 해당 페이지만 INACTIVE. 다른 공개 페이지는 유지됨.

에이전트는 반드시 어떤 의도인지 확인 후 처리해야 함.

## 5. 이미지 갤러리 ("한 행에 이미지 여러개")

- 이미지는 sequence 번호로 그룹핑
- 같은 sequence에 기본 이미지 1개 + 추가 이미지 N개 가능
- 기본 이미지: isAlwaysPublic: true (상시)
- 추가 이미지: startAt~endAt 기간에만 표시 (같은 자리에 날짜별로 다른 이미지)
- 추가 이미지끼리 날짜 겹침 불가
- 기본 이미지 삭제: 추가 이미지 모두 삭제 후에만 가능

## 6. 결제수단 — 상품유형별

### 상품유형별 결제수단 매핑

| 결제수단 | 정기구독 (SUBSCRIPTION) | 일시결제 (ONE_TIME) | 선물하기 | 응원하기 |
|----------|:---:|:---:|:---:|:---:|
| 신용카드 | O | O | O | O |
| 카카오페이 | O (자동결제) | O | O | O |
| 네이버페이 | X | O | O | O |
| 삼성페이 | X | O | O | O |
| 가상계좌 | X | O | X | X |

### 결정 로직
- `product.paymentInfo.type === 'SUBSCRIPTION'` → 카드 + 카카오페이 (2개)
- `product.paymentInfo.type === 'ONE_TIME_PURCHASE'` → 카드 + 네이버 + 삼성 + 카카오 + 가상계좌 (5개)
- 선물하기 → 일시결제에서 가상계좌 제외 (4개)
- 응원하기 → 일시결제에서 가상계좌 제외 (4개)
