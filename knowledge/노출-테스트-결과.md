# 노출 테스트 결과 (2026-03-31)

## 테스트 환경
- 세팅 에이전트: localhost:8000
- QA 에이전트: localhost:8001
- us-plus: localhost:3000 (web-mcp 브랜치)

## 오피셜클럽 publicType별 노출

테스트 대상:
- PUBLIC: 조조형우 (cmsId=35, productGroupCode=145)
- PRIVATE: FE 테스트 오피셜클럽 (cmsId=129)

| 영역 | PUBLIC (cmsId=35) | PRIVATE (cmsId=129) |
|------|-------------------|---------------------|
| 구독 탭 카드 | ✅ 보임 (order=11, 버튼 활성) | ❌ 안 보임 |
| 마스터 추천 탭 | ✅ 보임 | ❌ 안 보임 |
| 클럽 페이지 URL 직접 접근 | ✅ 접근 가능, 멤버십 배너 보임 | ✅ **접근 가능**, 멤버십 배너 보임 |
| 나의 마스터 피드 (/feed) | ✅ 보임 (팔로우/구독 중) | ❌ 안 보임 |

### 핵심 발견

1. **PRIVATE = 목록에서 숨김, URL 직접 접근은 허용**
   - 구독 탭, 마스터 추천 탭에서 안 보임 (서버가 API 응답에서 제외)
   - 하지만 `/club/{masterId}` URL로 직접 접근하면 **정상 렌더링됨**, 멤버십 배너도 보임
   - 히든 상품페이지(isHidden)와 동일한 패턴
   - publicType 필드는 클라이언트 API 응답에 없음 — 서버가 필터링 후 내려줌

2. **cmsId ≠ productGroupCode**
   - 조조형우: cmsId=35, productGroupCode=145
   - 구독 탭 카드 조회 시 productGroupCode 사용 필요
   - `check_subscribe_club`은 cmsId로 조회 가능 (Web MCP에서 매핑)

3. **나의 마스터 피드는 팔로우/구독 기반**
   - PRIVATE이어도 팔로우 중이면 보일 수 있음 (현재 129는 팔로우 안 해서 안 보임)
   - PUBLIC이어도 팔로우 안 하면 안 보임

## 기존 확인 완료 (R0~R7, 2026-03-28)

상품페이지/상품 레벨:

| 규칙 | 내용 |
|------|------|
| R0 | 메인 상품페이지 EXCLUDED → 구독 버튼 안 보임 |
| R1 | 상품페이지 INACTIVE → URL 접근 시 "공개되지 않은 상품 페이지" |
| R2 | 상품페이지 히든 → 구독 탭에 안 보이지만 URL 직접 접근 가능 |
| R3 | 비히든 날짜지정(기간내) > 상시 — 상시 페이지를 비활성화 |
| R4 | 히든 = 노출 경쟁에서 제외 — 기간 충돌 검사 제외 |
| R5 | 모든 상품 비공개 → 상품페이지는 뜨지만 상품 안 보임 |
| R6 | 구독탭 웹링크는 isProductGroupWebLinkStatus ACTIVE여야 반영 |
| R7 | 히든 해제 시 기간 충돌 검사 미적용 (생성 시에만) |

## 신규 규칙 (2026-03-31)

| 규칙 | 내용 |
|------|------|
| R8 | 오피셜클럽 PRIVATE → 구독 탭, 마스터 추천 탭에서 안 보임 (서버 필터링) |
| R9 | 오피셜클럽 PRIVATE → 클럽 페이지 URL 직접 접근은 가능 (히든과 동일 패턴) |
| R10 | 나의 마스터 피드는 팔로우/구독 기반이지만, PRIVATE 전환 시 피드에서도 사라짐 |
| R11 | cmsId ≠ productGroupCode — 구독 탭 조회 시 productGroupCode 필요 |
| R12 | 메인 상품페이지 EXCLUDED → 구독 탭에서 **카드 자체가 사라짐** (버튼만 비활성이 아님) |
| R13 | 메인 상품페이지 EXCLUDED → 클럽 페이지 멤버십 배너도 사라짐 |
| R14 | 메인 상품페이지 EXCLUDED → 클럽 페이지 접근 자체는 가능 (콘텐츠만 보임) |
