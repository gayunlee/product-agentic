# 노출 테스트 결과 (2026-03-31)

## 테스트 환경
- 세팅 에이전트: localhost:8000
- QA 에이전트: localhost:8001
- us-plus: localhost:3000 (web-mcp 브랜치)
- 테스트 대상: FE 테스트 오피셜클럽 (cmsId=129, PRIVATE)

## 오피셜클럽 publicType별 노출

| 영역 | PUBLIC | PRIVATE | 비고 |
|------|--------|---------|------|
| 구독 탭 (멤버십 구독) | 보임 | **안 보임** | 서버가 PRIVATE 마스터를 API 응답에서 제외 |
| 마스터 추천 탭 | 보임 | **안 보임** | 동일 — API에서 제외 |
| 나의 마스터 피드 (/feed) | 미확인 | 미확인 | Web MCP tool 미구현 |
| 클럽 페이지 (/club/{id}) | 접근 가능 | **접근 불가** (렌더링 안 됨) | PRIVATE면 컴포넌트 마운트 안 됨 |

### 핵심 발견
- **PRIVATE 오피셜클럽은 서버가 API 응답에서 아예 제외** — 프론트 필터링이 아님
- publicType 필드 자체가 클라이언트 API 응답에 없음 — 서버가 필터링 후 내려줌
- 클럽 페이지 직접 접근 시 PRIVATE면 렌더링 실패 (에러 페이지 또는 빈 페이지)

## 기존 확인 완료 (R0~R7, 2026-03-28)

상품페이지/상품 레벨:

| 규칙 | 내용 |
|------|------|
| R0 | 메인 상품페이지 EXCLUDED → 구독 버튼 안 보임 |
| R1 | 상품페이지 INACTIVE → URL 접근 시 "공개되지 않은 상품 페이지" |
| R2 | 상품페이지 히든 → 구독 탭에 안 보이지만 URL 직접 접근 가능 |
| R3 | 비히든 날짜지정(기간내) > 상시 — 상시 페이지를 비활성화 |
| R4 | 히든 = 노출 경쟁에서 제외 — 기간 충돌 검사 제외 |
| R5 | 모든 상품 비공개 → 상품페이지는 뜨지만 상품 안 보임 |
| R6 | 구독탭 웹링크는 isProductGroupWebLinkStatus ACTIVE여야 반영 |
| R7 | 히든 해제 시 기간 충돌 검사 미적용 (생성 시에만) |

## 미확인 항목

| 항목 | 필요한 것 |
|------|----------|
| 나의 마스터 피드 (/feed) | Web MCP `get_my_master_feed` tool |
