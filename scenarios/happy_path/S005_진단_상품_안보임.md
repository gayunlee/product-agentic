# S005. 진단: 상품 페이지가 안 보여

## 메타데이터
- **카테고리**: happy_path
- **난이도**: medium
- **필요 권한**: PRODUCT + MASTER
- **관련 Phase**: 진단

## 사용자 입력
> "조조형우 상품 페이지가 고객한테 안 보이는데 왜 그런지 확인해줘"

## 기대 행동

에이전트는 "안 보여" 키워드를 인식하고 진단 모드로 진입한다.
아래 체크리스트를 **순서대로** 확인하고, 원인이 발견되면 해결 방법을 안내한다.

### 진단 체크리스트
1. 마스터 공개 상태 확인
   - Tool: `search_masters(search_keyword="조조형우")`
   - 확인: publicType이 "PUBLIC"인지?
   - PENDING/PRIVATE이면 → "마스터가 PUBLIC 상태가 아닙니다. 전환이 필요합니다."

2. 메인 상품 페이지 활성화 상태 확인
   - Tool: `get_master_detail(master_id="{cmsId}")`
   - 확인: productGroupViewStatus가 "ACTIVE"인지?
   - INACTIVE/EXCLUDED이면 → "메인 상품 페이지가 활성화되지 않았습니다."

3. 상품 페이지 공개 상태 확인
   - Tool: `get_product_page_list(master_id="{cmsId}")`
   - 확인: 상품 페이지의 status가 "ACTIVE"인지?
   - INACTIVE이면 → "상품 페이지가 비공개 상태입니다."

4. 공개 기간 확인
   - 상품 페이지의 startAt/endAt 확인
   - 오늘이 기간 밖이면 → "공개 기간이 아닙니다. 기간: {startAt} ~ {endAt}"

5. 상품 옵션 노출 확인
   - Tool: `get_product_list_by_page(product_page_id="{pageId}")`
   - 확인: isDisplay가 true인 옵션이 있는지?
   - 모두 false이면 → "노출 중인 상품 옵션이 없습니다."

6. 원인 종합 보고 + 해결 방법 제안
   - 발견된 원인별로 해결 방법 안내
   - "ACTIVE로 전환할까요?" 같은 액션 제안

## 기대 응답
- 진단 결과 요약 (어떤 항목이 문제인지)
- 해결 방법 구체적 안내
- 필요한 경우 "전환할까요?" 같은 액션 제안
- 관리자센터 링크

## 절대 하면 안 되는 것
- 체크리스트 순서를 건너뛰기 (1번부터 순서대로 확인)
- 원인 확인 없이 바로 상태 변경
- "잘 모르겠습니다" 같은 모호한 응답 (반드시 체크리스트 결과를 보고)

## 평가 기준
- [ ] "안 보여" 키워드 인식 → 진단 모드 진입
- [ ] 체크리스트 순서대로 확인 (마스터 공개 → 메인 상품 → 페이지 공개 → 기간 → 옵션 노출)
- [ ] 조회 Tool만 사용 (생성/수정 Tool 사용 금지, 진단 단계에서)
- [ ] 원인 발견 시 구체적 해결 방법 안내
- [ ] 관리자센터 링크 포함
