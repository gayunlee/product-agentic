# SP001. 구독상품 전체 세팅 (사이드패널)

## 메타데이터
- **카테고리**: happy_path
- **난이도**: hard
- **필요 권한**: PRODUCT + MASTER
- **모드**: 사이드패널 (네비게이터)

## 사용자 입력
> "조조형우 마스터로 구독상품 세팅하고 싶어"

## 사이드패널 플로우

### Step 1: 마스터 확인 (API 직접)
```
사이드패널: search_masters("조조형우") + get_master_groups("조조형우")
→ "✅ 조조형우 (PUBLIC, cmsId:35)"
→ 자동으로 Step 2로
```

### Step 2: 시리즈 확인 (API 직접)
```
사이드패널: get_series_list("35")
→ "✅ 시리즈 2개: 투자 기초 시리즈, 경제 분석 리포트"
→ "사전 조건 충족! 상품 페이지를 생성할 수 있습니다."

표시:
  [📄 상품 페이지 생성하러 가기]  → /product/page/create
  💡 "마스터 선택에서 '조조형우'를 선택해주세요"
  [✅ 상품 페이지 생성 완료]
```

### Step 3: 상품 페이지 생성 확인 (유저가 페이지에서 직접)
```
유저: [✅ 상품 페이지 생성 완료] 클릭
사이드패널: get_product_page_list("35")
→ "✅ 상품 페이지 확인"
→ "  페이지명: {title}, 코드: {code}, 상태: 비공개"

표시:
  "이미지를 등록해주세요."
  [🖼️ 이미지 설정으로 이동]  → /product/page/{id}?masterId=35&tab=settings
  [⏭️ 이미지는 나중에]
  [✅ 이미지 등록 완료]
```

### Step 4: 이미지 등록 (유저가 페이지에서 직접)
```
유저: [✅ 이미지 등록 완료] 또는 [⏭️ 나중에]
→ Step 5로
```

### Step 5: 상품 옵션 등록 (유저가 페이지에서 직접)
```
사이드패널:
  "상품 옵션을 등록해주세요."
  [📦 상품 옵션 등록하러 가기]
    → /product/create?productPageId={id}&productType=SUBSCRIPTION&masterId=35

  💡 "입력 항목: 상품명, 금액, 결제주기, 포함할 시리즈, 프로모션(선택)"
  [✅ 상품 옵션 등록 완료]
  [➕ 옵션 하나 더 등록]

유저: [✅ 완료] 클릭
사이드패널: get_product_list_by_page("{id}")
→ "✅ 상품 옵션 1개 확인: 월간 투자 리포트 29,900원/월"
```

### Step 6: 활성화 (API 직접)
```
사이드패널:
  "세팅 완료! 활성화하면 고객에게 노출됩니다."
  ⚠️ 이미지 플레이스홀더면: "이미지가 아직 등록되지 않았습니다."
  [🚀 활성화하기]

유저: [🚀 활성화하기] 클릭
사이드패널:
  → update_product_display (노출 ON)
  → update_product_sequence (순서)
  → update_product_page_status (ACTIVE)
  → update_main_product_setting (ACTIVE)
→ "✅ 활성화 완료!"
```

### Step 7: 검증 + 체크리스트 (API 직접)
```
사이드패널: get_product_page_detail + get_product_list_by_page
→ "검증 결과:"
→ "  ✅ 상품 페이지 ACTIVE"
→ "  ✅ 메인 상품 페이지 ACTIVE"
→ "  ✅ 구독 탭 {N}번째 노출"

  "⚠️ 체크리스트:"
  [🖼️ 이미지 설정]  → ?tab=settings
  [⚠️ 유의사항 등록]  → ?tab=caution
  [🌐 고객 화면 확인]  → https://dev.us-insight.com/products/group/{code}
```

## 버튼 호출 순서
1. [📄 상품 페이지 생성하러 가기] → 유저 이동 → [✅ 완료]
2. [🖼️ 이미지 설정] → 유저 이동 → [✅ 완료] or [⏭️ 나중에]
3. [📦 상품 옵션 등록] → 유저 이동 → [✅ 완료]
4. [🚀 활성화하기] → 사이드패널 API 직접
5. [🌐 고객 화면 확인] → 유저 확인

## 절대 하면 안 되는 것
- 상품 페이지를 API로 직접 생성 (페이지로 이동시켜야 함)
- 상품 옵션을 API로 직접 등록 (페이지로 이동시켜야 함)
- [완료] 없이 다음 단계 진행 (재확인 필수)
- 이미지를 API로 업로드 시도

## 평가 기준
- [ ] Step 순서 준수 (1→2→3→4→5→6→7)
- [ ] 생성/등록은 페이지 이동 버튼 제공 (API 직접 호출 안 함)
- [ ] [완료] 후 API로 재확인
- [ ] 활성화는 API 직접 (4단계)
- [ ] 검증 자동 수행 + 체크리스트 안내
