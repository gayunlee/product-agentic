# #2 상품페이지 수정할려고

## 분류
- 유저 의도: 수행 (guide_edit_page)
- 에이전트: 오케스트레이터 → 수행 에이전트 + 검증 에이전트

## 사전조건 체인
```
오피셜클럽(cmsId) → 상품 페이지 특정(productPageId)
```

## 에이전트 역할
- 오케스트레이터: "수정할려고" → 수행 요청으로 분류
- 검증 에이전트: check_prerequisites("get_product_page_detail", context)
- 수행 에이전트: 오피셜클럽 → 상품 페이지 선택 → 수정 페이지 이동 안내

---

## Case A: 정상 플로우

```
유저: 상품페이지 수정할려고
에이전트: 어떤 오피셜클럽의 상품 페이지를 수정하시겠어요?
유저: 조조형우
에이전트: (search_masters → get_product_page_list)
         조조형우 상품 페이지 3개 있습니다:
         1. 월간 투자 리포트 (코드: 148, 구독, 공개)
         2. 단건 특강 (코드: 155, 단건, 비공개)
         3. 프리미엄 멤버십 (코드: 160, 구독, 공개, 히든)
         어떤 페이지를 수정하시겠어요?
유저: 1번
에이전트: 상품 페이지 수정 화면으로 이동해주세요.
         [상품 페이지 수정으로 이동 →] (/product/page/{id}?tab=settings)
```

## Case B: 상품 페이지 없음

```
유저: 상품페이지 수정할려고
에이전트: 어떤 오피셜클럽의 상품 페이지를 수정하시겠어요?
유저: 김영익
에이전트: (search_masters → get_product_page_list)
         김영익 오피셜클럽에 상품 페이지가 없습니다.
         새로 생성하시겠어요?
         [페이지 생성으로 이동 →] (/product/page/create)
```

## Case C: 상품 페이지 1개 → 바로 안내

```
유저: 상품페이지 수정할려고
에이전트: 어떤 오피셜클럽의 상품 페이지를 수정하시겠어요?
유저: 봄날의 햇살
에이전트: (search_masters → get_product_page_list)
         봄날의 햇살 상품 페이지 1개 있습니다:
         - 단건상품 (코드: 124, 단건, 공개)
         이 페이지를 수정하시겠어요?
유저: ㅇㅇ
에이전트: [상품 페이지 수정으로 이동 →] (/product/page/{id}?tab=settings)
```

## Case D: 컨텍스트에 이미 정보가 있는 경우

```
(사이드패널 URL: /product/page/123?tab=settings)
유저: 상품페이지 수정할려고
에이전트: 현재 보고 계신 상품 페이지를 수정하시겠어요?
유저: ㅇㅇ
에이전트: [상품 페이지 수정으로 이동 →] (/product/page/123?tab=settings)
```

---

## 검증 포인트 (eval 기준)

1. 오피셜클럽 특정 없이 수정 시도 → 반드시 질문
2. 상품 페이지 여러 개 → 유저에게 선택 요청 (임의 선택 금지)
3. 상품 페이지 없음 → 생성 안내로 전환
4. 컨텍스트에 productPageId 있으면 확인 후 바로 안내
