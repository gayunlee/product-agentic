# #16 URL로만 접근가능하고 다른데서 안 보이게 하고 싶은데

## 분류
- 유저 의도: 수행 (guide_hidden_setup)
- 에이전트: 오케스트레이터 → 수행 에이전트
- 방식: API 직접 실행 또는 수정 페이지 이동 (유저 선택)

---

## Case A: 기존 페이지 히든 처리

```
유저: URL로만 접근가능하고 다른데서 안 보이게 하고 싶은데
에이전트: 히든 처리를 하시면 됩니다.
         기존 페이지를 히든으로 변경하시겠어요, 새로 만드시겠어요?
유저: 기존 페이지
에이전트: 어떤 오피셜클럽의 상품 페이지인가요?
유저: 조조형우
에이전트: (목록 조회)
         1. 월간 투자 리포트 (코드: 148, 공개)
         2. 단건 특강 (코드: 155, 비공개)
         어떤 페이지를 히든 처리하시겠어요?
유저: 1번
에이전트: 월간 투자 리포트를 히든 처리하시겠어요?
         히든 처리 시 구독 탭/오피셜클럽 홈에서 사라지고 URL로만 접근 가능합니다.
         [히든 처리하기] [직접 수정하기 →] (/product/page/{id}?tab=settings)
유저: (히든 처리하기 클릭)
에이전트: 히든 처리 완료했습니다.
         [상품 페이지 확인하기 →] (/product/page/{id}?tab=settings)
```

```
에이전트: 히든 처리를 하시면 됩니다.
         기존 페이지를 히든으로 변경하시겠어요, 새로 만드시겠어요?
유저: 새로 만들려고
에이전트: (→ #1 상품페이지 생성 플로우로 전환, 히든=true로 안내)
```

## 검증 포인트 (eval 기준)

1. 기존/신규 구분 질문
2. API 직접 실행 버튼 + 직접 수정 페이지 이동 버튼 둘 다 제공
3. 히든 처리 전 유저 확인 필수
4. 완료 후 확인 페이지 이동 버튼 제공
