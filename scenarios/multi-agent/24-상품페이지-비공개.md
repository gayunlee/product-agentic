# #24 상품페이지 비공개해줘

## 분류
- 유저 의도: 수행 (toggle_page_status)
- 에이전트: 오케스트레이터 → 수행 에이전트
- 방식: API 직접 실행 또는 수정 페이지 이동 (유저 선택)

## 사전조건 체인
```
오피셜클럽(cmsId) → 상품 페이지 특정
```

---

## Case A: 정상

```
유저: 상품페이지 비공개해줘
에이전트: 어떤 오피셜클럽의 상품 페이지인가요?
유저: 조조형우
에이전트: (목록 조회)
         1. 월간 투자 리포트 (코드: 148, 공개)
         2. 단건 특강 (코드: 155, 비공개)
         어떤 페이지를 비공개 하시겠어요?
유저: 1번
에이전트: 월간 투자 리포트를 비공개 처리하시겠어요?
         비공개 시 고객에게 페이지가 보이지 않습니다.
         [비공개 처리하기] [직접 수정하기 →] (/product/page/{id}?tab=settings)
유저: (비공개 처리하기)
에이전트: 비공개 처리 완료했습니다.
         [상품 페이지 확인하기 →] (/product/page/{id}?tab=settings)
```

## Case B: 의도 확인 — 구독 페이지에서 아예 안 보이게 vs 특정 페이지만 비공개

```
유저: 상품페이지 안 보이게 해줘
에이전트: 어떤 오피셜클럽인가요?
유저: 봄날의 햇살
에이전트: 어떤 방식으로 안 보이게 하시겠어요?

         1. 구독 페이지에서 아예 안 보이게
            → 메인 상품페이지를 EXCLUDED 처리합니다. 구독 탭에서 봄날의 햇살 버튼이 사라지고
              모든 상품페이지에 접근할 수 없게 됩니다.

         2. 특정 상품페이지만 비공개
            → 선택한 상품페이지만 비공개(INACTIVE) 처리합니다. 다른 공개 페이지가 있으면
              그 페이지는 계속 노출됩니다.

유저: 1번
에이전트: 봄날의 햇살의 메인 상품페이지를 EXCLUDED 처리하시겠어요?
         구독 페이지에서 봄날의 햇살이 완전히 사라집니다.
         [EXCLUDED 처리하기] [메인 상품 페이지 관리 →]

유저: (EXCLUDED 처리하기)
에이전트: EXCLUDED 처리 완료. 구독 페이지에서 봄날의 햇살이 안 보입니다.
```

```
유저: 상품페이지 안 보이게 해줘
유저: 봄날의 햇살
유저: 2번 (특정 상품페이지만)
에이전트: (목록 조회)
         1. 단건상품 (코드: 124, 공개)
         2. 테스트 (코드: 155, 공개)
         어떤 페이지를 비공개 하시겠어요?
유저: 1번
에이전트: 단건상품을 비공개 처리하시겠어요?
         [비공개 처리하기] [직접 수정하기 →]
```

## Case C: 마지막 공개 페이지 비공개 (계단식 경고)

```
유저: 상품페이지 비공개해줘
에이전트: 어떤 오피셜클럽의 상품 페이지인가요?
유저: 봄날의 햇살
에이전트: (목록 조회 → 공개 페이지 1개뿐)
         "단건상품" (코드: 124, 공개) ← 마지막 공개 페이지
         비공개 처리하시겠어요?

         ⚠️ 이 페이지가 마지막 공개 페이지입니다.
         비공개 처리하면 모든 상품페이지가 비공개됩니다.
         메인 상품페이지도 EXCLUDED 처리하지 않으면 구독 페이지에서 버튼 클릭 시 404가 발생합니다.

         [비공개 처리하기] [메인도 EXCLUDED 처리] [직접 수정하기 →]
유저: (메인도 EXCLUDED 처리)
에이전트: 비공개 처리 + 메인 EXCLUDED 완료했습니다.
```

## 검증 포인트 (eval 기준)

1. "상품페이지" = 상품 페이지 (상품 옵션이 아님)
2. 오피셜클럽 → 상품 페이지 특정
3. API 실행 버튼 + 직접 수정 버튼 둘 다 제공
4. 변경 전 유저 확인 필수
5. 완료 후 확인 페이지 이동
