# 버그 확인 사항

## BUG-1: PRIVATE 오피셜클럽 URL 직접 접근 가능 (R9)

**현상:**
- 오피셜클럽 publicType=PRIVATE인데 `/club/{masterId}` URL로 직접 접근하면 정상 렌더링됨
- 멤버십 배너도 보임, 클럽 콘텐츠도 보임
- 구독 탭, 마스터 추천 탭, 피드에서는 안 보임 (정상)

**예상 동작:**
- PRIVATE이면 URL 직접 접근도 차단되어야 하는 게 아닌지? (비공개 상품페이지 INACTIVE는 "공개되지 않은 상품 페이지입니다"로 차단됨)
- 아니면 히든 상품페이지처럼 의도적으로 URL 접근은 허용하는 건지?

**재현:**
- FE 테스트 오피셜클럽 (cmsId=129, PRIVATE)
- `http://localhost:3000/club/129/contents` 접속 → 정상 렌더링

**확인 필요:**
- [ ] 기획 의도 확인 — PRIVATE 클럽 URL 직접 접근이 의도된 건지
- [ ] 프로덕션에서도 동일한지 확인
- [ ] 백엔드 API가 PRIVATE 클럽 데이터를 반환하는지 (클라이언트 렌더링은 데이터가 오면 그냥 보여주니까)
