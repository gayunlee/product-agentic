- **난이도: 낮음 (프론트만)**

### 3. UX 개선 — 설계 필요

#### 3-1. 투기적 실행 (Speculative Execution) ★ 핵심

**개념**: 모호한 요청이 오면 가능한 분기를 병렬로 미리 실행하고, 유저 확인 후 맞는 결과만 사용.

**예시 플로우**:
```
유저: "상품페이지 비공개 해줘"

기존 (순차):
  LLM: "메인 상품페이지인가요, 일반 상품페이지인가요?" → 대기 3s
  유저: "일반 상품페이지"
  LLM: search_masters → 대기 3s
  LLM: get_product_page_list → 대기 3s
  총: ~9s 대기

투기적 실행 (병렬):
  LLM: "어떤 걸 말하시나요?" 질문 전송 (즉시)
  동시에: search_masters + get_product_page_list 미리 조회 시작
  유저: "일반 상품페이지" 답변
  → 이미 조회된 결과에서 골라서 바로 표시
  총: ~3s 대기
```

**검토 필요 사항**:
- executor LLM이 "모호함 감지 → 병렬 조회" 판단을 어떻게 하는가
- 불필요한 API 호출 비용 vs 속도 개선 트레이드오프
- Strands SDK에서 병렬 tool 실행 지원 여부 (Haiku 4.5는 sequential)
- 병렬 조회 결과 캐싱/폐기 전략
- 프론트에서 "조회 중..." → "결과 선택" 전환 UX
